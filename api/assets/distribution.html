Accepted workbook formats

CCS Chemical Register

The standard CCS template. The parser reads:

  • Customer info from cells C8, C10, C12
  • Products from rows 16–120 (columns A & B)
  • Column N must be marked — put Yes, 1, or ✓ to include a product
  • SDS and Risk Assessment files are matched by product code from the server's document library
Structured / Client Workbook

Any sheet with clearly labelled columns. The parser auto-detects header rows containing:

  • Customer sheet: email, company, name
  • Products sheet: product code or product name + sds url / sds link
  • Optional: risk assessment url, chemical register url, selected

Upload the client workbook

Select an .xlsx or .xlsm file, then click Preview workbook. The preview will show parsed customer info, each product's SDS and Risk Assessment links, and flag any missing documents.

Tip: for the CCS Chemical Register format, make sure column N is marked Yes or 1 for each product you want to include.

Compatibility Notice

This tool was built specifically for the CCS Chemical Register format — standard column layout with product codes in column A, product names in column B, and active products marked in column N. Workbooks that match that layout will parse reliably.

Other workbook formats — custom layouts, non-standard column orders, or workbooks without a clearly marked selection column — may not parse correctly. If no products are found, the preview panel will flag what to check.

Password-protected or encrypted workbooks will not work — remove workbook protection before uploading.

To add support for a new workbook format, email a sample to support@nxai.com.au and we'll build it in.
